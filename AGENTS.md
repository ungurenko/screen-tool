# AGENTS.md — локальные правила работы с ScreenTool

## Как общаться с Александром
- Всегда отвечай на русском языке.
- Пиши просто: сначала результат, потом короткие детали.
- Используй уместные эмодзи для структуры.
- Александр не программист, поэтому объясняй бытовым языком.
- Если нужно дать команду пользователю, дай одну копируемую строку.

## Статус проекта
- Это локальная версия ScreenTool, взятая из открытого исходного кода.
- Проект отвязан от исходного GitHub-репозитория автора.
- Рабочий remote `origin` указывает на репозиторий Александра `ungurenko/screen-tool`.
- Не добавляй обратно исходный upstream автора без прямой просьбы Александра.
- Не публикуй релизы, issues, PR или GitHub Actions в чужой репозиторий.
- Лицензию и упоминания AGPL-3.0 не удаляй без отдельного юридического решения.

## Текущие технические решения
- Git remote `origin` указывает на `ungurenko/screen-tool`.
- Метаданные `package.json` больше не указывают на чужой GitHub.
- Публикация `electron-builder` в GitHub Releases отключена.
- Автообновления выключены по умолчанию.
- Для будущего собственного канала обновлений нужны `SCREENTOOL_UPDATE_FEED_URL` и `SCREENTOOL_ENABLE_AUTO_UPDATES=1`.

## Как работать с кодом
- Меняй только то, что относится к текущей просьбе.
- Перед заметной правкой проверь реальные файлы и поведение проекта.
- Для поиска по проекту сначала используй `rg`.
- Для UI делай чистый светлый интерфейс и проверяй, что текст помещается.
- Не трогай секреты, сертификаты, `.p12`, локальные записи и `.screentool` файлы.

## Быстрые проверки
- `npm run lint` — проверка форматирования и правил кода.
- `npm test` — быстрые тесты.
- `npm run i18n:check` — проверка переводов.
- Для отдельных тестов используй `npx vitest --run path/to/file.test.ts`.

## Что важно помнить
- Ребрендинг на ScreenTool уже выполнен; новые проекты сохраняются в формате `.screentool`.
- Если задача касается релиза или обновлений, сначала настрой собственный репозиторий и канал доставки.
- Перед коммитом всегда смотри `git status` и коммить только изменения текущей задачи.
